## 一、产品定位

| 维度 | 说明 |
|------|------|
| **名称** | InkVerse |
| **Slogan** | Where Stories Become Universes. |
| **定位** | AI 驱动的沉浸式小说创作平台。AI 与作者共同创作故事宇宙。 |

## 五、Story Canvas（核心）

### 交互模型

| 操作 | 触发 | 反馈 |
|------|------|------|
| **滚轮缩放** | Mouse Wheel | 画布缩放（0.15x ~ 3x），以光标位置为中心 |
| **平移画布** | 空格 + 拖拽 / 中键拖拽 | 光标变为 grab/grabbing |
| **拖动节点** | 鼠标拖拽节点 | 节点跟随，松手时回弹+入历史栈 |
| **点击节点** | 单击 | 右侧弹出节点详情面板（玻璃拟态） |
| **编辑节点** | 双击节点 / 详情面板点编辑 | 行内编辑标题+描述，Enter保存 Esc取消 |
| **删除节点** | 详情面板点删除 | 节点+相关连线移除，入历史栈 |
| **创建节点** | 画布空白处双击 | 弹出类型选择，创建新节点 |
| **连接节点** | 右键节点→连接 | 进入连线模式 |
| **右键菜单** | 右键节点 | 编辑/复制/删除/连接 |
| **撤回/重做** | CTRL+Z / CTRL+SHIFT+Z | 50步历史栈 |
| **删除选中节点** | Delete 键 | （选中的节点） |
| **聚焦节点** | 侧栏点击列表项 | 画布平滑移动到该节点位置 |
| **侧栏筛选** | 点击人物/章节等 | 画布仅显示对应类型节点，底部显示筛选条 |

### 节点设计

| 类型 | 颜色 | 边框色 | 图标 | 尺寸 |
|------|------|------|------|------|
| 人物 | #E0E7FF | #A5B4FC | 👤 | 220×≥110px |
| 章节 | #EDE9FE | #C4B5FD | 📖 | 220×≥110px |
| 地点 | #D1FAE5 | #6EE7B7 | 📍 | 220×≥110px |
| 事件 | #FFF7ED | #FDBA74 | ⚡ | 220×≥110px |
| 伏笔 | #FEE2E2 | #FCA5A5 | 🔮 | 220×≥110px |

- 圆角：18px
- 左侧4px色带标识类型
- AI生成节点：紫色glow + "AI"标签
- 悬停：scale 1.03 + 阴影增强
- 点击：scale 0.98
- 新节点：Fade + Scale 弹簧入场

## 六、AI 创作工作台

右侧320px固定面板。

### 5个Tab
| Tab | 建议内容 |
|------|---------|
| 📝 剧情建议 | 冲突转折、副本设计、情感节点 |
| 👤 人物建议 | 角色深化、关系发展、背景填补 |
| 📖 章节建议 | 下一章方向、POV选择、衔接方式 |
| 🔮 伏笔建议 | 铺设→发酵→回收生命周期管理 |
| 🌍 世界观 | 系统规则、场景设计、历史背景 |

### 交互
- 建议卡片：标题+描述+"+ 插入画布"按钮
- 插入画布：以AI生成节点形式出现在Canvas上（紫色glow）
- 底部聊天输入：快速向AI提问

## 十一、HCI创新点总结

1. **Story Canvas** —— 将小说结构从线性文本转化为空间化节点网络，降低长文本的认知负荷
2. **混合主动交互** —— 三种模式（AI主导/协同/手动）动态切换，用户控制粒度可变
3. **Agent思考过程可见** —— SSE流式输出+分步展示，让AI从黑盒变为可理解的协作伙伴
4. **玻璃拟态+微交互** —— 现代设计语言降低工具的"技术感"，让创意工作者感到亲切
5. **撤回+历史** —— 50步操作历史，降低AI辅助创作中的"怕出错"焦虑
